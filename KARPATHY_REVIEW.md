## 1. HARDCODED DATA FOUND

### 1.1 trends_analyzer.py - Fake Search Volumes
**File**: `trends_analyzer.py:327-337`
**Issue**: Hardcoded base search volumes that are not real data
```python
base_volumes = {
    'solar': 15000,
    'pool': 20000,
    'plumbing': 25000,
    ...
}
```
**Problem**: These are made-up numbers, not actual search volume data
**Solution**: Use pytrends actual interest scores OR integrate with a real search volume API (SEMrush, Ahrefs, or Google Keyword Planner)

### 1.2 leads_output/index.html - All Lead Data Hardcoded
**File**: `leads_output/index.html:816-905`
**Issue**: Every lead's SEO score, ranking, and opportunity data is hardcoded JavaScript
```javascript
{
    seo_score: 45,
    ranking_position: 1,
    opportunity: { total_monthly_searches: "12,000+", ... }
}
```
**Problem**: Dashboard shows fake data, not connected to actual analysis
**Solution**: Load data from database or JSON files generated by the Python workflow

### 1.3 trends_analyzer.py - Location Codes Limited
**File**: `trends_analyzer.py:34-41`
**Issue**: Only 6 locations supported
```python
LOCATION_CODES = {
    'los angeles, ca': 'US-CA',
    'austin, tx': 'US-TX',
    ...
}
```
**Problem**: System only works for LA and Austin, not expandable
**Solution**: Use a geocoding API or maintain a comprehensive location database

## 2. FAILURE MODES FROM MECHANISM LOG

### 2.1 Firecrawl Search/Scrape Failures
**Current**: No retry logic, no error handling
**Solution**: Add exponential backoff, retry 3 times, log failures

### 2.2 PageSpeed API Rate Limits
**Current**: Shows "rate_limited" but no recovery
**Solution**: Cache results, implement queue with delays, get API key

### 2.3 pytrends Rate Limits
**Current**: 429 errors, fallback to estimated data
**Solution**: Cache aggressively (24hrs), queue requests, respect rate limits

### 2.4 Contact Extraction Failures
**Current**: Misses emails in images, JavaScript, obfuscated patterns
**Solution**: Add headless browser rendering, more regex patterns, manual verification

### 2.5 Database Failures
**Current**: No concurrent access handling
**Solution**: Use WAL mode, connection pooling, proper error handling
